Hermes Agent

Guia completo do Dockerfile para o assistente de terminal da Nous Research.

Voltar

Este guia descreve como o container Docker está estruturado para rodar de forma segura o Hermes Agent da Nous Research em um ambiente isolado. O Hermes é um assistente de IA que roda diretamente no seu terminal, com suporte a múltiplos provedores via OpenRouter.

Dockerfile Original

FROM ubuntu:latest

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && apt-get install -y \
    curl \
    ca-certificates \
    sudo \
    git \
    xz-utils \
    && rm -rf /var/lib/apt/lists/*

RUN curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

WORKDIR /workspace

CMD ["bash"]

Explicação Passo a Passo

1. Imagem Base

FROM ubuntu:latest: Utiliza a versão estável mais recente do Ubuntu como ponto de partida. O Ubuntu oferece compatibilidade total com ferramentas de desenvolvimento, shell scripting e gerenciamento de pacotes.

2. Modo Não Interativo

ENV DEBIAN_FRONTEND=noninteractive: Suprime todos os prompts e diálogos interativos durante a instalação de pacotes APT. Essencial para builds automatizadas sem intervenção humana.

3. Instalação das Dependências

O comando RUN executa os seguintes passos:

  • curl — baixa arquivos da internet (usado para o script de instalação do Hermes).
  • ca-certificates — certificados SSL para conexões HTTPS seguras.
  • sudo — elevação de privilégios para comandos administrativos.
  • git — controle de versão, essencial para o Hermes interagir com repositórios.
  • xz-utils — descompactação de arquivos .xz usados em distribuições binárias.

O rm -rf /var/lib/apt/lists/* ao final remove o cache do APT para manter a imagem final mais enxuta.

4. Instalação do Hermes Agent

RUN curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash: Baixa e executa o script de instalação oficial do Hermes. As flags -fsSL garantem silêncio, seguimento de redirects e falha em caso de erro HTTP.

5. Diretório de Trabalho

WORKDIR /workspace: Define a pasta padrão onde o container inicia. É neste diretório que você monta os arquivos do seu projeto com -v $(pwd):/workspace.

6. Comando Padrão

CMD ["bash"]: Inicializa um shell Bash interativo ao subir o container. A partir dele você pode executar hermes setup ou hermes diretamente.

Comandos de Execução

Use estes comandos para rodar o Hermes Agent no seu workspace atual:

Construir Imagem
docker build -t hermes-agent ./hermes
Executar Container Interativo
docker run -it --rm -v $(pwd):/workspace hermes-agent
Executar com API Key e iniciar o Hermes direto
docker run -it --rm -e OPENROUTER_API_KEY="sk-or-..." -v $(pwd):/workspace hermes-agent hermes

Configuração do Hermes

Depois que o container estiver rodando, configure o acesso aos modelos de IA. O Hermes usa o OpenRouter como provedor padrão.

Método 1: Setup Interativo

O assistente de configuração guia você na escolha do provedor, chave de API e modelo preferido.

Comando no Terminal (dentro do container)
hermes setup
Método 2: Variável de Ambiente

Se você já tem uma chave do OpenRouter, exporte-a como variável de ambiente e inicie o Hermes diretamente.

Comandos no Terminal
export OPENROUTER_API_KEY="sk-or-v1-...aqui"
hermes
Método 3: Configuração Direta

Configure o modelo e a chave de API diretamente via CLI, sem o assistente interativo.

Comandos no Terminal
hermes config set model anthropic/claude-opus-4.6
hermes config set OPENROUTER_API_KEY sk-or-...token
hermes

Parâmetros do docker run

Opção Significado
-it Modo interativo com TTY — você digita comandos dentro do container
--rm Remove o container automaticamente quando você sair
-e OPENROUTER_API_KEY="..." Passa sua chave da API do OpenRouter como variável de ambiente
-v "$(pwd):/workspace" Monta a pasta atual do seu PC dentro do container em /workspace

Dicas e Solução de Problemas

Problema Solução
"docker: command not found" Docker não está instalado. Instale o Docker Desktop ou siga o guia oficial.
"permission denied" ao rodar docker No Linux: sudo usermod -aG docker $USER e faça logout/login.
"cannot connect to model" Verifique se a chave OPENROUTER_API_KEY está correta e com créditos.
Build muito lento Normal na primeira vez. Builds seguintes usam cache do Docker.
Arquivos não aparecem no /workspace Confira se usou -v "$(pwd):/workspace" na pasta correta.
Modelo não encontrado no /model Use /model dentro do Hermes para listar e escolher modelos disponíveis.

Como obter uma chave de API (OpenRouter)

  1. Acesse openrouter.ai
  2. Crie uma conta gratuita
  3. Vá em Settings → Keys
  4. Clique em Create Key
  5. Copie a chave (formato sk-or-v1-...)
