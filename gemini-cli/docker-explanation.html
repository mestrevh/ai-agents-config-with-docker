Explicação do Dockerfile

Este documento explica como o arquivo Dockerfile está configurado para construir a imagem deste ambiente.

Conteúdo Original

FROM ubuntu

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && apt-get install -y \
    curl \
    gnupg \
    && curl -fsSL https://deb.nodesource.com/setup_20.x | bash - \
    && apt-get install -y nodejs \
    && rm -rf /var/lib/apt/lists/*

RUN npm install -g @google/gemini-cli

WORKDIR /workspace

CMD ["bash"]

Passo a Passo

1. Imagem Base

FROM ubuntu: Utiliza a versão mais recente do Ubuntu como sistema operacional base.

2. Configuração de Ambiente

ENV DEBIAN_FRONTEND=noninteractive: Define uma variável de ambiente para que as instalações de pacotes não solicitem intervenção do usuário (prompts).

3. Instalação do Node.js

O comando RUN executa uma série de operações:

  • Atualiza a lista de pacotes (apt-get update).
  • Instala dependências básicas (curl, gnupg).
  • Configura o repositório da NodeSource para o Node.js versão 20.x.
  • Instala o nodejs.
  • Limpa os arquivos temporários do cache para reduzir o tamanho da imagem.
4. Instalação do Gemini CLI

RUN npm install -g @google/gemini-cli: Instala globalmente a interface de linha de comando do Gemini utilizando o gerenciador de pacotes do Node.js.

5. Diretório de Trabalho

WORKDIR /workspace: Define /workspace como o diretório onde os comandos serão executados por padrão dentro do container.

6. Comando Padrão

CMD ["bash"]: Inicia o shell bash automaticamente quando o container é iniciado.
