Claude Code

Explicação detalhada do Dockerfile para o assistente de terminal da Anthropic.

Voltar

Este guia descreve como o container Docker está estruturado para rodar de forma segura o Claude Code da Anthropic em um ambiente isolado.

Dockerfile Original

FROM ubuntu:latest

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && apt-get install -y \
    curl \
    gnupg \
    ca-certificates \
    git \
    && curl -fsSL https://deb.nodesource.com/setup_20.x | bash - \
    && apt-get install -y nodejs \
    rm -rf /var/lib/apt/lists/*

RUN npm install -g @anthropic-ai/claude-code

WORKDIR /workspace

CMD ["bash"]

Explicação Passo a Passo

1. Imagem Base

FROM ubuntu:latest: Utiliza a versão estável mais recente do Ubuntu. Oferece compatibilidade total com ferramentas de desenvolvimento e controle de versão.

2. Modo Não Interativo

ENV DEBIAN_FRONTEND=noninteractive: Suprime prompts e diálogos interativos durante a instalação de pacotes no Ubuntu.

3. Instalação do Node.js e Dependências

O comando RUN executa os seguintes passos essenciais:

  • Instala ferramentas necessárias como curl, gnupg e ca-certificates.
  • Instala o git, que é um requisito essencial para o Claude Code interagir com repositórios e gerenciar modificações.
  • Adiciona o repositório oficial do Node.js 20.x da NodeSource.
  • Instala o nodejs e remove caches do apt para manter a imagem o mais enxuta possível.
4. Instalação Global do Claude Code

RUN npm install -g @anthropic-ai/claude-code: Utiliza o gerenciador de pacotes do Node (NPM) para fazer o download e instalação global da CLI oficial do Claude Code.

5. Diretório de Trabalho

WORKDIR /workspace: Configura a pasta default de inicialização de comandos do container.

6. Execução Padrão

CMD ["bash"]: Inicializa o console Bash para permitir o uso interativo do terminal ao subir o container.

Comandos de Execução

Use estes comandos para rodar o Claude Code no seu workspace atual:

Construir Imagem
docker build -t claude-code ./claude-code
Executar Container Interativo
docker run -it --rm -v $(pwd):/workspace claude-code

Uso com Chaves de API Externas

Como utilizar o Claude Code com provedores alternativos (OpenRouter, Gemini, DeepSeek, GPT-4o).

Opção 1: OpenRouter (Rápido e Simples)

O OpenRouter unifica vários provedores de IA em uma única chave de API. Ele é diretamente compatível com o Claude Code porque podemos mudar a URL de destino da API.

Como fazer: Dentro do container Docker, configure a URL base e a sua chave do OpenRouter antes de iniciar o Claude:

Comandos no Terminal
export ANTHROPIC_BASE_URL="https://openrouter.ai/api/v1"
export ANTHROPIC_API_KEY="sk-or-sua-chave-openrouter"
claude --model google/gemini-2.5-pro
  • ANTHROPIC_BASE_URL redireciona as chamadas para o OpenRouter.
  • ANTHROPIC_API_KEY deve ser a sua chave do OpenRouter (iniciando em sk-or-).
  • Substitua o modelo no comando (ex: google/gemini-2.5-pro, deepseek/deepseek-chat).
Opção 2: LiteLLM (Proxy Tradutor Universal)

O LiteLLM age como um intermediário local. Ele traduz as mensagens que o Claude Code envia (formato da Anthropic) para a linguagem de outros provedores como Google Gemini, OpenAI ou DeepSeek automaticamente.

Passo 1: Instale o LiteLLM (no seu ambiente local ou dentro do container)
Instalação via PIP
pip install litellm
Passo 2: Crie o arquivo de configuração litellm-config.yaml

Este arquivo mapeia os modelos de outros provedores para que o Claude Code os encontre: 

model_list:
  # Google Gemini
  - model_name: gemini-2.5-pro
    litellm_params:
      model: gemini/gemini-2.5-pro
      api_key: os.environ/GEMINI_API_KEY

  # OpenAI GPT
  - model_name: gpt-4o
    litellm_params:
      model: openai/gpt-4o
      api_key: os.environ/OPENAI_API_KEY

  # DeepSeek
  - model_name: deepseek-chat
    litellm_params:
      model: deepseek/deepseek-chat
      api_key: os.environ/DEEPSEEK_API_KEY
Passo 3: Suba o Proxy local informando sua chave do provedor escolhido
Iniciar Proxy local
export GEMINI_API_KEY="sua-chave-api-aqui"
litellm --config litellm-config.yaml --port 4000
Passo 4: Aponte o Claude Code para o proxy local

Abra uma nova janela de terminal, exporte o endereço do proxy local (porta 4000) e execute o Claude:

Executar Claude Code apontando para o LiteLLM
export ANTHROPIC_BASE_URL="http://localhost:4000"
export ANTHROPIC_API_KEY="qualquer-coisa" # Requisitado pelo CLI, mas ignorado pelo LiteLLM
claude --model gemini-2.5-pro
