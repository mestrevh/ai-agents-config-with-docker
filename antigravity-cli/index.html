O que é um Dockerfile?

A receita de bolo automatizada para a infraestrutura da sua aplicação.

Arquivo de Texto

Um simples arquivo de texto chamado Dockerfile (sem extensão) contendo comandos ordenados que descrevem o ambiente.

Infraestrutura como Código

Permite rastrear mudanças de dependências, versões de sistemas operacionais e configurações em sistemas de controle de versão como Git.

Camadas Imutáveis

Cada instrução gera uma nova camada imutável no cache do Docker. Se nada mudou, o Docker reaproveita as camadas rapidamente.

Anatomia de um Dockerfile

Clique em cada linha do código abaixo para ver o que cada instrução faz no seu container.

Dockerfile
FROM ubuntu:latestENV DEBIAN_FRONTEND=noninteractiveRUN apt-get update && apt-get install -y \
    curl \
    ca-certificates \
    sudo \
    && rm -rf /var/lib/apt/lists/*RUN curl -fsSL https://antigravity.google/cli/install.sh | bashWORKDIR /workspaceCMD ["bash"]
FROM

Imagem Base

Define a imagem mãe sobre a qual seu container será construído. Quase todo Dockerfile precisa começar com a instrução FROM.

Dica de Ouro

Prefira sempre usar tags de versão específicas como ubuntu:22.04 ou imagens otimizadas para produção (como Alpine Linux) em vez de latest, para garantir builds consistentes.

Como Executar o Dockerfile

Do código ao container rodando. Entenda o ciclo de compilação (Build) e execução (Run).
